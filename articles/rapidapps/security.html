Security

We recently decided to change our default security policy when using Rapp. From now on, the first time you publish an application using wn push, it will disallow root logins using SSH:

...
remote: Security: Preventing root login...
remote: 
remote: --------------------------------------------------------------
remote:  
remote:  IMPORTANT: Rapp now will disable root SSH access by default. 
remote: 
remote:  If you want to overwrite this setting follow this steps:
remote: 
remote:  1. Make sure you have the latest gem version (1.0.4)
remote:  
remote:     $ wn version
remote:     [Webbynode] Rapid Deployment Gem v1.0.4
remote: 
remote:  2. If needed, update your gem
remote: 
remote:     $ gem install webbynode
remote: 
remote:  3. Set the allow_root setting to true
remote: 
remote:     $ wn settings add allow_root true
remote:  
remote:  4. Run 'wn push' and root will be allowed again.
remote:  
remote: --------------------------------------------------------------
...

The steps above sums up what you need to do if you want to reestablish root login, but we advise you don’t. Instead of using the root user, you can safely use the git user that is created specially for the Rapp engine.

This user has sudo privileges, so you can do anything you used to do with root.

To login using git, just use ssh normally:

$ ssh git@100.200.10.1

If you deployed a Rapp application to your server before, it’s likely that a password won’t be prompted, since we use the safer public key authorization.
